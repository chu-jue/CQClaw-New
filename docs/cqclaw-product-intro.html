CQClaw 是一个 AI 驱动的 Android 自动化测试平台。它既提供可视化界面，让开发和测试人员低成本编排自动化流程；也提供 CLI 能力，让 AI Agent 观察真实设备、执行动作、验证结果并沉淀脚本。

可复用 一次任务沉淀为后续可执行 Profile

核心价值

面向开发和测试团队，CQClaw 把传统高成本的 Android 自动化，转化为可视化、可执行、可沉淀的工作流。

1

降低自动化门槛

把安装、启动、点击、输入、截图、日志检查等步骤变成可视化工作流，减少脚本编写成本。

2

让 AI 操作手机

通过 cqclaw CLI，Agent 可以观察真实设备、执行动作、验证结果，并保存为复用流程。

3

提升排查效率

日志洞察、节点解析、截图、Dump 和运行证据集中管理，帮助团队更快复现与定位问题。

4

沉淀测试资产

一次人工或 AI 执行的任务可以保存为 Profile，后续回归测试直接复用。

5

创造团队价值

减少手工回归和问题复现成本，提升自动化覆盖率、问题定位效率和交付速度。

产品界面

核心功能都围绕“操作手机、理解页面、分析日志、管理证据、复用流程”展开。

自动化编排

面向用户和 Agent 的统一流程编辑器，支持预览、执行、保存与复用。

日志洞察

实时 Logcat、文件导入、过滤、Crash/ANR 定位和规则化问题识别。

节点解析

将 UIAutomator Dump 转换为可查询节点，帮助 Agent 选择稳定的操作目标。

资源中心

集中预览和管理截图、Dump、日志、APK、临时脚本与工作流导出。

架构亮点

同一套本地服务同时服务 Web、桌面客户端和 AI CLI，让人的操作和 Agent 的操作互相复用。

人和 AI 共用同一套手机自动化能力

开发和测试人员通过 Web 或桌面客户端低成本编排流程；AI Agent 通过 cqclaw CLI 执行观察、点击、输入、截图、日志检查和文件操作；本地服务统一管理设备、证据和工作流 Profile。

使用场景

从日常自测到团队回归，从日志排查到 AI 自动化执行，都可以用同一个平台完成。

回归测试

将高频流程沉淀为可重复执行的 Profile。

开发自测

快速验证安装、启动、页面跳转和核心功能。

问题复现

保留截图、日志和步骤证据，复现过程更稳定。

日志排查

导入日志或实时采集，定位 Crash、ANR 和关键事件。

多设备验证

统一管理设备和运行结果，降低手工切换成本。

AI 自动化

让 Agent 调用 CLI 操作手机并沉淀流程。

CQClaw 将手机自动化从“写脚本”升级为“可视化编排 + AI 执行 + 资产复用”。

它帮助团队以更低的人力和技术成本提升测试覆盖率、问题定位效率和交付速度，让开发、测试和 AI Agent 共享同一套可持续演进的 Android 自动化能力。
